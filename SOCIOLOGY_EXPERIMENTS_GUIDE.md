# 社会学议题实验模板指南

本文档介绍 4 个社会学议题的实验设计，这些模板内置了预配置的代理和参数，用于探索社会科学中的核心现象。

---

## 1. 社会规范的突变 (Ethnomethodology)

### 理论基础
- **常人方法论** (Ethnomethodology) - Garfinkel
- **社会规范的形成与崩溃** - 关注群体如何通过互动构建、维持或推翻规范

### 实验配置

| 参数 | 值 |
|-----|-----|
| 场景类型 | `norm_disruption` |
| 代理数量 | 20 个 |
| 代理构成 | 异质背景（10 种职业 × 2）+ 10 种性格 |
| 核心属性 | 规范遵守度、社会地位、反抗倾向 |
| 干预变量 | 引入"违反常理"的新规范（例：禁止下午说话、强制物物交易） |

### 观察重点

1. **意义构建 (Sense-making)**
   - 代理如何理解并解释突发规范？
   - 是否进行集体协商来重新定义规范？

2. **适应策略**
   - 选择服从（高规范遵守度的代理）
   - 选择反抗（高反抗倾向的代理）
   - 私下形成"黑市"协议（特定职业群体的联盟）

3. **戈夫曼拟剧论**
   - 公开表演 vs 后台行为
   - 代理在公众场合是否隐藏真实态度？

### 关键指标

```
规范遵守度 = 0~100
  - 高（80+）：倾向公开遵守新规范
  - 低（<30）：容易挑战或规避规范

反抗倾向 = 0~100
  - 高（70+）：可能发起反抗运动
  - 低（<30）：容易被同化
```

### 建议干预步骤

1. **第 1-2 轮**：宣布新规范
   - 操作：`public_broadcast` - "从今日起，禁止下午 3 点后在公共场合说话"

2. **第 3-5 轮**：观察适应过程
   - 记录各代理的意见表达频率
   - 观察是否形成反抗声音

3. **第 6-10 轮**：黑市协议形成
   - 跟踪私下对话频率
   - 检测是否有群体开始规避规范

## 2. 政策传播中的"意义磨损" (Street-level Bureaucracy)

### 理论基础
- **街头官僚制** (Street-level Bureaucracy) - Michael Lipsky
- **信息异化** - 官僚制中政策变形
- **国家视阈下的底层逻辑** - 詹姆斯·斯科特

### 实验配置

| 参数 | 值 |
|-----|-----|
| 场景类型 | `policy_diffusion` |
| 组织结构 | 三层：政府(3) → 社区(7) → 居民(10) |
| 政策类型 | 模糊性高的政策（如"优化资源分配促进公平"） |
| 核心属性 | 官级、灵活性、个人利益驱动、现实理解度 |

### 层级特征

#### 政府层（Policy Makers）
- 官级：90（最高权力）
- 意识形态强度：80（信念坚定）
- 现实理解度：30（远离基层）

#### 社区层（Street-level Bureaucrats）
- 官级：50（中等权力）
- 灵活性：60~90（高适应性）
- 个人利益驱动：40（利益冲突）

#### 居民层（Policy Recipients）
- 官级：10（无权力）
- 实用性：80（关注实际收益）
- 对官僚的不信任度：50~90

### 观察重点

1. **政策重构过程**
   - 政府发布模糊政策 → 社区主管根据个人利益重新解读 → 居民根据本地现实继续变形

2. **意义磨损轨迹**
   ```
   政府版本：
   "优化资源分配以促进公平"

   社区版本：
   "根据当地情况灵活分配，优先照顾关键部门"

   居民版本：
   "社区主管根本不在乎，资源都被关系户拿走了"
   ```

3. **利益冲突表现**
   - 中层官员是否优先照顾亲密关系？
   - 基层居民是否对政策产生不信任？

### 关键对话框架

**政府 → 社区主管**
```
"请执行政策XX，确保资源公平分配"
```

**社区主管 → 居民**
```
"根据实际情况，资源主要分配给..."
（可能加入个人偏好）
```

**居民反馈**
```
"这哪里是公平？分明是偏心！"
```

### 建议干预步骤

1. **第 1 轮**：政府发布政策
   - 操作：`public_broadcast` - 政府版本政策

2. **第 2-4 轮**：社区层级传达
   - 观察社区主管如何改写政策
   - 记录他们对居民的不同解释

3. **第 5-10 轮**：居民反应与信任侵蚀
   - 追踪对政府和官员的不信任度增长
   - 检测是否产生反抗或绕过政策的行为

## 3. 极化与"数字茧房"生成 (Echo Chamber Effect)

### 理论基础
- **舆论极化** (Polarization)
- **回声壁效应** (Echo Chamber Effect)
- **群体极化** (Group Polarization) - Cass Sunstein
- **情绪传染** (Emotional Contagion)

### 实验配置

| 参数 | 值 |
|-----|-----|
| 场景类型 | `polarization` |
| 代理构成 | 激进派(8) + 保守派(8) + 中立派(4) = 20 |
| 推荐机制 | "推荐算法"Agent 基于阅读历史推送内容 |
| 核心属性 | 立场强度、情绪易激怒度、包容度、接收度 |

### 代理群体特征

#### 激进派（8 个）
- 立场强度：85~100（不可动摇）
- 情绪易激怒度：70~95（容易被触发）
- 包容度：20~40（不容忍异见）
- 接收度：0（完全拒绝对立信息）

#### 保守派（8 个）
- 立场强度：75~95（相对坚定）
- 情绪易激怒度：60~90
- 包容度：30~50
- 接收度：100（更容易接收信息）

#### 中立派（4 个）
- 立场强度：30~60（容易摇摆）
- 情绪易激怒度：40~70
- 包容度：60~90（开放心态）
- 接收度：50（选择性接收）

### 推荐算法机制

```
for each 代理 in 社交网络:
  if 代理.last_5_articles 都是[立场A]:
    推荐更多[立场A]的内容 (强化)
  else if 代理.包容度 > 70:
    推荐[对立立场]的内容 (尝试平衡)
  else:
    推荐[相同立场]的内容 (茧房)
```

### 观察重点

1. **极化加速**
   - 中立派是否逐渐投向某一极端？
   - 激进派的立场强度是否继续上升？

2. **茧房形成**
   - 代理是否主要接收与己方观点一致的信息？
   - 不同立场群体之间的对话是否减少？

3. **情绪传染**
   - 跟踪"情绪易激怒度"随时间的变化
   - 观察高激怒度的代理是否传染给周围人

4. **推荐算法的影响**
   - 禁用推荐算法，观察对话是否更开放？
   - 启用算法，观察极化速度是否加快？

### 关键指标

```
群体极化指数 =
  (激进派平均立场强度 - 保守派平均立场强度)
  / 初始差异

情绪激怒指数 =
  对立方观点出现时的回复中性词% 下降
  / 总回复数
```

### 建议干预步骤

1. **第 1-2 轮**：基线测量
   - 记录各派别的初始立场强度和包容度

2. **第 3-5 轮**：启用推荐算法
   - 观察推荐如何改变信息流
   - 追踪各派别的立场强度变化

3. **第 6-10 轮**：极化过程
   - 记录不同派别之间的对话减少
   - 检测中立派向某一极端倾斜

4. **对照实验**
   - 禁用推荐算法，重新运行
   - 比较有无算法下的极化速度

## 4. 稀缺资源压力下的社会契约演化 (Hobbesian Jungle)

### 理论基础
- **霍布斯丛林** (Hobbesian State of Nature)
- **社会资本** (Social Capital) - Bourdieu, Putnam
- **互惠利他** (Reciprocal Altruism)
- **长期记忆与信用** (Reputation & Trust)

### 实验配置

| 参数 | 值 |
|-----|-----|
| 场景类型 | `resource_scarcity` |
| 背景设定 | 灾后恢复期社区 |
| 稀缺资源 | 食物、清水（有限且递减） |
| 代理构成 | 20 个，按社会资本分层 |
| 核心属性 | 社会资本、诚实度、绝望指数、欺诈倾向 |

### 代理分类

#### 高信用代理（6 个，社会资本 70~100）
- 职业：医生、教师、工程师
- 社会资本来源：职业声誉
- 行为倾向：倾向遵守协议
- 欺诈倾向：10~30%

#### 中等信用代理（7 个，社会资本 40~60）
- 职业：农民、工人、商贩
- 社会资本来源：邻里关系
- 行为倾向：取决于绝望指数
- 欺诈倾向：30~60%

#### 低信用代理（7 个，社会资本 10~30）
- 职业：黑市商人、陌生人
- 社会资本来源：欠缺
- 行为倾向：更易欺诈
- 欺诈倾向：60~100%

### 核心机制

#### 长期记忆（Vector DB 模拟）

每个代理记住：
```
{
  交互对手: 名字,
  事件类型: [帮忙 | 背叛 | 公平交易],
  记忆强度: 时间衰减,
  信任评分: -100 ~ +100
}
```

#### 信用契约流程

```
阶段1: 初期接触
  高信用代理主动提出合作 → 资源共享协议

阶段2: 互惠验证
  是否有人违反承诺？
  → 是：信任崩溃 → 转入敌对模式
  → 否：信任强化 → 继续合作

阶段3: 群体协议
  是否形成"信用联盟"？
  或演变成"黑市"？
```

### 观察重点

1. **信用契约是否形成**
   - 高信用代理能否成为"信用中心"？
   - 中等信用代理是否加入或创建替代网络？

2. **欺诈与崩溃**
   - 是否有低信用代理进行大规模欺诈？
   - 欺诈是否导致信用体系崩溃？
   - 高信用代理如何应对背叛？

3. **替代机制的形成**
   - "黑市"是否形成（资源交易不经官方）？
   - 强制分配是否导致暴力冲突？

4. **社会分化**
   - 是否形成"信用阶层"和"被排斥者"？
   - 是否有贫富分化加剧的现象？

### 关键指标

```
社会凝聚度 =
  总合作协议数 / 理论最大协议数

信用体系稳定性 =
  1 - (背叛事件数 / 总互动数)

资源不平等系数 =
  (最富有代理资源量 - 最贫困代理资源量)
  / 社区平均资源量
```

### 建议干预步骤

1. **第 1 轮**：初始状态
   - 宣布资源总量与分配规则
   - 记录各代理的初始社会资本

2. **第 2-4 轮**：协议形成期
   - 观察高信用代理的主动协议提议
   - 追踪不同群体的协议互动

3. **第 5-7 轮**：压力测试
   - 逐渐减少资源（模拟灾害恶化）
   - 观察信用体系是否开始崩溃
   - 检测欺诈行为的出现

4. **第 8-10 轮**：终局状态
   - 评估最终的社会分化程度
   - 统计有多少人陷入完全孤立
   - 检测是否形成暴力冲突

    // 资源实验
    社会资本?: 0-100

    欺诈倾向?: 0-100

### 推荐模式（实现建议）

对于"极化"场景，在场景中添加一个虚拟"推荐算法"Agent，逻辑如下：

```python
def recommendation_algorithm(agents, conversation_history):
    for agent in agents:
        recent_content = conversation_history[agent.id][-5:]
        stance_distribution = analyze_stance(recent_content)

        if stance_distribution['same_camp'] > 0.8:
            # 强化效应：推荐更多同立场内容
            recommend_content(agent, stance='same')
        elif agent.properties['包容度'] > 70:
            # 包容代理：尝试平衡信息
            recommend_content(agent, stance='mixed')
        else:
            # 默认：推荐同立场
            recommend_content(agent, stance='same')
```

### 长期记忆模拟（资源实验）

```python
def remember_interaction(agent, counterparty, event_type, outcome):
    memory_entry = {
        'counterparty': counterparty.id,
        'event': event_type,  # 'help', 'betray', 'trade'
        'outcome': outcome,    # success / failure
        'timestamp': current_turn,
        'weight': 1.0
    }

    agent.memory.append(memory_entry)

    # 信任评分更新
    if event_type == 'help' and outcome == 'success':
        agent.trust[counterparty.id] += 20
    elif event_type == 'betray':
        agent.trust[counterparty.id] -= 50
```

## 运行示例

### 社会规范的突变

```
第 1 轮：宣布新规范
干预：public_broadcast
内容："从今日起，下午 3 点后禁止在公共场合说话"

预期观察：
- 高规范遵守度代理：接受
- 高反抗倾向代理：质疑

第 3 轮：观察黑市形成
记录：是否有代理开始私下交流？
```

### 政策传播的意义磨损

```
第 1 轮：政府宣布
政策官员说："优化资源分配以促进公平"

第 2 轮：社区传达
社区主管对居民说："根据实际情况，资源优先分配给..."
（可能根据个人利益调整）

第 5 轮：居民反应
居民："政府的政策根本没有被执行！"
```

### 极化与茧房

```
第 1 轮：基线
激进派平均立场强度：88
保守派平均立场强度：82

启用推荐算法

第 10 轮：结果
激进派平均立场强度：95
保守派平均立场强度：91
中立派：已转向激进派或保守派
```

### 资源稀缺下的契约

```
初期：高信用医生主动提议：
"我们建立互相帮助的网络，保证粮食和水的公平分配"

中期：欺诈出现
黑市商人暗中囤积资源，背叛协议

后期：社会分化
形成"信用联盟"和"被排斥者"两个群体
```

## 扩展方向

1. **情感维度**：添加情绪状态（恐惧、愤怒、希望），观察其如何驱动行为
2. **信息流分析**：追踪假信息如何在网络中传播
3. **制度干预**：测试不同制度设计（透明化、惩罚机制、激励）对结果的影响
4. **跨实验关联**：组合多个实验（如"规范变化 + 政策传播"），观察复杂社会现象
